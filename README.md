# Pharmacy_databasesystem
A database to manage pharmacy, inventory, suppliers, staff, customers and purchases in a structured form.

This is a database desiged to maintain a strucured data in Pharmacy. 
This project checks for  data consistency, reduces errors, and improves operational efficiency.
It also keeps the accurate purchase recording.


This database contains 6 entities which are:
                                             1. Pharmacy
                                             2. Customer
                                             3. Supplier
                                             4. Staff
                                             5. Purchases
                                             6. Inventory


RELATIONSHIPS IN THE DBMS:
                          One supplier supplies many medicines
                          One customer can make multiple purchases
                          One staff member handles multiple purchases
                          Each medicine is tracked in inventory


The ER diagram representing the database structure is included in this repository.
                          

Technologies used:
                   1. My SQL 
                   2. My SQL Workbench
